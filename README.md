# Апдейкон — навигатор обновлений 1С

Веб-сервис для построения цепочек обновлений конфигураций 1С:Предприятие.
Отвечает на вопрос «с версии X на версию Y — какие `.cfu`-файлы накатить и в каком порядке».

База данных содержит **603+ конфигурации** и **200 000+ рёбер** графа обновлений,
автоматически синхронизируется с `downloads.v8.1c.ru`.

## Быстрая установка

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/iMironRU/updatecon/main/deploy.sh)
```

Скрипт:
- проверяет ОС (Ubuntu 20.04+ / Debian 11+)
- устанавливает Docker если отсутствует
- запрашивает логин и пароль для административной панели
- предлагает восстановить готовый дамп данных (603 конфиг, >200k обновлений) —
  сервис сразу работает без ИТС-аккаунта
- показывает в консоли адрес сайта и админки

По завершении в консоли:


## Что умеет

- **Калькулятор цепочки** — выбираете конфигурацию, текущую и целевую версию,
  получаете упорядоченный список `.cfu`-файлов со ссылками на releases.1c.ru
- **Каталог** — 603+ конфигурации с датами выхода версий, минимальной платформой,
  частотой обновлений
- **Поиск по версии** — узнайте, к каким конфигурациям относится версия
- **Административная панель** (`/admin`) — статус БД, логи импорта, ручной
  запуск синхронизации с ИТС, настройки расписания

## Требования

| | |
|---|---|
| ОС | Ubuntu 20.04 / 22.04 / 24.04 или Debian 11 / 12 |
| RAM | от 1 ГБ (минимум 512 МБ, но во время импорта тесно) |
| Диск | от 5 ГБ (Docker-образы ~800 МБ + OS + данные) |
| Порт | 3000 (настраивается через `WEB_PORT` в `.env`) |

Docker и Docker Compose устанавливаются автоматически при их отсутствии.

## Ручная установка (без curl | bash)

```bash
git clone https://github.com/iMironRU/updatecon.git
cd updatecon
cp .env.example .env
# Отредактируйте .env: задайте POSTGRES_PASSWORD, ADMIN_LOGIN, ADMIN_PASSWORD
nano .env
docker compose build
docker compose up -d
```

## Конфигурация (.env)

| Переменная | Описание | По умолчанию |
|---|---|---|
| `ADMIN_LOGIN` | Логин административной панели | `admin` |
| `ADMIN_PASSWORD` | Пароль административной панели | `changeme` |
| `ITS_LOGIN` | Логин портала 1С (для авто-синхронизации) | — |
| `ITS_PASSWORD` | Пароль портала 1С | — |
| `IMPORT_CRON` | Расписание импорта (cron) | `0 4 * * *` |
| `IMPORT_ON_START` | Импорт при старте воркера | `1` |
| `WEB_PORT` | Внешний порт сайта | `3000` |

> **Важно:** `ITS_LOGIN` и `ITS_PASSWORD` никогда не попадают во фронтенд —
> только в окружение воркера на сервере.

## Обновление данных

Без ИТС-кредов данные берутся из дампа (`data/seed.sql.gz`).
Если задать `ITS_LOGIN` / `ITS_PASSWORD` в `.env` и перезапустить воркер,
синхронизация будет происходить автоматически по расписанию `IMPORT_CRON`.

Принудительный запуск через админку: `/admin` → вкладка «Импорт».

## Управление

```bash
docker compose logs -f worker   # прогресс импорта
docker compose logs -f web      # веб-сервер
docker compose restart web      # применить изменения .env для веба
docker compose down             # остановить (данные сохраняются в volume)
docker compose down -v          # остановить и удалить данные
```
